Image Bundler Tool for jQuery + NodeJS

Let's face it, surfing sites on mobile devices on a 3G network is slow. Even when images are cached, they still do http responses of 304s (not modified). How do we increase performance? We reduce http requests by turning images into base64 encoded strings! This project was inspired by @rem's inliner tool; however, it now lets developers specify which images they want to base64 encode by adding the attribute "cid-src='/myimage.jpg'".

Cid is a client (uses a bit of jQuery for swapping cid-src -> src + jsonp) and server (NodeJS for base64 encoding) tool for swapping in images for base64 encoded strings using Data URI Scheme.

How it works

Images I don't want base64 encoded!

This uses the regular src attribute.

Images I want to bundle

This uses 'cid-src' instead of 'src'









<img cid-src="public/images/Mog - Status.gif">
      <img cid-src="public/images/Mog - Credits.gif">
      <img cid-src="public/images/Mog (Front).gif">
      <img cid-src="public/images/Mog (Left).gif">

What Happened

  1. User goes to your product catalog site / image gallery on their mobile device (phone or tablet)
  2. JSONP request to the 'cid' server side image bundler
  3. Response comes back with a hash map of base64 encoded images (based on cid-src as key)
  4. Swap in cid-src for src on images to process them as base64 instead of an image http request (see Data URI Scheme)
